## 日期：2026-04-01 晚

### 修复Windows客户端日志不生成问题

修复内容：

1. **修复 NameError**: `Path` 导入顺序错误 - 在导入 `Path` 之前就使用了 `Path(__file__).parent`，导致启动失败，不生成日志。已经将 `from pathlib import Path` 移到文件开头。

2. **修复打包后日志丢失**: PyInstaller 打包单exe后，`Path(__file__)` 指向临时解压目录，程序退出后临时目录被删除，日志丢失。现在改为检测 `sys.frozen`，打包模式下使用 `sys.executable` 所在目录作为基础目录，日志永久保存在 exe 目录下的 `cache/client.log`。

3. **日志策略调整**: 按要求只记录失败，不记录成功：
   - 心跳失败 → 记录日志，心跳成功不输出
   - 解密失败 → 记录日志，密钥续期成功不输出
   - 删除所有失败信息的 `print`，只写日志文件
   - 删除了"密钥已更新，有效期至..."刷屏信息

修复后：
- Windows 启动客户端不再崩溃
- 日志文件 `cache/client.log` 正常生成
- 只有错误才记录，成功操作不产生日志，保持日志干净
- 即使打包成 exe，日志也不会丢失，方便问题排查
